# GDoc Fixer 제품 설명서

## 1. 제품 개요

### 1.1 제품 소개

GDoc Fixer는 HTML 문서 편집 및 AI 기반 문서 자동화 웹 애플리케이션입니다.
DOCX, HWP, XLSX 등 다양한 포맷의 문서를 임포트하여 HTML로 변환하고,
Google Gemini AI를 활용해 문서 수정, 프레젠테이션 생성, 차트 렌더링 등을 수행합니다.

### 1.2 핵심 가치

| 항목 | 설명 |
|------|------|
| **AI 기반 자동화** | 자연어 명령으로 문서 수정, 슬라이드 생성, 차트 삽입 |
| **다양한 포맷 지원** | DOCX, HWP, XLSX, 이미지, PDF 임포트 / PNG, DOCX, PDF, HTML 내보내기 |
| **대용량 문서 처리** | 4MB 이상 DOCX 문서도 스타일 압축 + DIFF 모드로 안정적 처리 |
| **온프레미스 배포** | PostgreSQL + 로컬 스토리지로 사내 서버 운영 가능 |

### 1.3 기술 스택

| 구분 | 기술 | 버전 |
|------|------|------|
| **프론트엔드** | React + Vite + Tailwind CSS | 19.x / 7.x / 4.x |
| **상태 관리** | Zustand | 5.x |
| **백엔드** | Express.js + Prisma ORM | 5.x / 6.x |
| **데이터베이스** | PostgreSQL | 15+ |
| **인증** | Passport.js + JWT | - |
| **파일 스토리지** | 로컬 파일시스템 (NFS/SAN) | - |
| **AI 엔진** | Google Gemini (2.5 Pro / 2.5 Flash / 3.1 Flash Image) | REST API |
| **차트** | Chart.js | 4.x |

## 2. MS Copilot 대비 Gemini API 채택 장점

본 제품은 Microsoft Copilot 대신 Google Gemini API를 AI 엔진으로 채택하였습니다.
아래는 문서 편집 솔루션 관점에서의 비교 분석입니다.

### 2.1 비교 요약

| 비교 항목 | Google Gemini API | MS Copilot |
|-----------|-------------------|------------|
| **컨텍스트 윈도우** | **1,048,576 토큰 (1M)** | 128,000 토큰 |
| **최대 출력 토큰** | **65,536 토큰** | 4,096~16,384 토큰 |
| **이미지 생성** | 내장 지원 (Gemini 3.1 Flash Image) | DALL-E 3 별도 호출 필요 |
| **멀티모달 입력** | 텍스트 + 이미지 + PDF + 오디오 + 비디오 | 텍스트 + 이미지 |
| **API 독립성** | REST API 직접 호출 (벤더 종속 없음) | Microsoft 365 생태계 종속 |
| **온프레미스 배포** | API 키만으로 즉시 사용 가능 | Azure AD + Microsoft 365 라이선스 필요 |
| **비용 구조** | 토큰 종량제 (무료 티어 제공) | Microsoft 365 구독 + Copilot 라이선스 |
| **모델 다양성** | Flash(빠른) / Pro(정밀) / Image(생성) 선택 | GPT-4 단일 모델 |

### 2.2 대용량 문서 처리 (핵심 차별점)

본 제품의 주요 사용 사례인 **대형 DOCX 문서 수정**에서 Gemini API의 1M 컨텍스트 윈도우는
결정적 이점을 제공합니다.

```
문서 크기별 처리 가능 여부:

                          Gemini API (1M 토큰)    MS Copilot (128K 토큰)
일반 문서 (~100KB)         ✅ FULL 모드             ✅ 처리 가능
중형 문서 (~500KB)         ✅ FULL 모드             ⚠️ 요약/분할 필요
대형 문서 (~1MB)           ✅ FULL 모드             ❌ 토큰 초과
초대형 문서 (~5MB)         ✅ DIFF 모드*            ❌ 토큰 초과
                          * 스타일 압축 후 처리 가능
```

- **Gemini**: 4.4MB DOCX 문서(인라인 스타일 포함)도 스타일 압축(74% 감소) 후
  1M 토큰 이내에서 처리 가능. 초대형 문서는 DIFF 모드로 변경분만 전송/수신.
- **Copilot**: 128K 토큰 한도로 인해 대형 문서는 분할 처리 또는 요약이 필수.
  원본 문서의 맥락을 유지한 채 수정하기 어려움.

### 2.3 멀티모달 파이프라인 (3-모델 전략)

Gemini API는 용도별 특화 모델을 제공하여, 단일 워크플로우 내에서
비용과 품질을 최적화할 수 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                  GDoc Fixer 3-모델 파이프라인                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Gemini 2.5 Flash ── 빠른 분석/파싱 ─── 저비용, 고속             │
│       ↓                                                         │
│  Gemini 3.1 Flash Image ── 이미지 생성 ─── 텍스트→이미지 내장     │
│       ↓                                                         │
│  Gemini 2.5 Pro ── 고품질 텍스트 생성 ─── 정밀 작업              │
│                                                                 │
│  ▶ 각 단계에 최적 모델을 배정하여 비용 절감 + 품질 극대화          │
│  ▶ MS Copilot은 GPT-4 단일 모델로 모든 작업 수행 (비용 비효율)    │
└─────────────────────────────────────────────────────────────────┘
```

| 작업 | GDoc Fixer (Gemini) | MS Copilot 방식 |
|------|---------------------|-----------------|
| 콘텐츠 분석 | Flash (저비용, ~0.1$/M토큰) | GPT-4 (고비용) |
| 이미지 생성 | Flash Image (내장, 추가 API 불필요) | DALL-E 3 별도 호출 + 비용 |
| 문서 조립 | Pro (정밀, 필요 시에만 사용) | GPT-4 (항상 고비용) |
| 뷰포트 보정 | Flash (스크린샷 분석) | 지원 없음 |

### 2.4 온프레미스 배포 용이성

| 항목 | Gemini API | MS Copilot |
|------|------------|------------|
| **필요 인프라** | API 키 1개 | Azure AD + Microsoft 365 E3/E5 + Copilot 라이선스 |
| **인증 연동** | 불필요 (서버 사이드 API 키) | Azure AD 필수 (SSO, 테넌트 설정) |
| **네트워크 요건** | HTTPS 아웃바운드만 허용 | Microsoft Graph API + Azure 다수 엔드포인트 허용 |
| **라이선스 비용** | API 호출 종량제만 | 사용자당 월 $30+ (Copilot 라이선스) |
| **데이터 주권** | API 호출 시만 데이터 전송, 저장 없음 | Microsoft 클라우드에 데이터 저장/처리 |

> **온프레미스 환경 결론**: Gemini API는 서버에서 REST 호출만 하면 되므로,
> 기존 인프라에 대한 변경 없이 즉시 도입 가능합니다.
> MS Copilot은 Microsoft 365 생태계 전체 구축이 선행되어야 합니다.

### 2.5 비용 비교 (월간 추정)

| 사용 규모 | Gemini API | MS Copilot |
|-----------|------------|------------|

| **50명, 일 10회 사용** | ~$50~150/월 (종량제) | $1,500/월 ($30/인 × 50명) |

## 3. 주요 기능

### 3.1 문서 편집기

HTML 코드 에디터와 실시간 미리보기를 좌우 분할 화면으로 제공합니다.

#### 기능 상세

| 기능 | 설명 |
|------|------|
| **코드 에디터** | HTML 소스 코드 직접 편집. 구문 강조 지원 |
| **실시간 미리보기** | 편집 내용을 iframe 기반으로 즉시 렌더링 |
| **분할 화면 조절** | 에디터/미리보기 간 드래그로 크기 조절 (200~1200px). 전체 화면 토글 지원 |
| **뷰포트 제어** | 너비/높이 직접 입력 또는 ±10px/±100px(Shift) 단위 조절 |
| **뷰포트 프리셋** | Mobile (375×667), Tablet (768×1024), Desktop (1920×1080), 사용자 정의 프리셋 저장/삭제 |

#### 동작 흐름

```
HTML 코드 입력 ──▶ Zustand Store 업데이트 ──▶ iframe 렌더링 ──▶ 실시간 미리보기
                                              │
                                              ▼
                                       뷰포트 크기 적용
                                   (반응형 디자인 확인 가능)
```

### 3.2 문서 임포트

다양한 포맷의 문서 파일을 HTML로 변환하여 편집기에 로드합니다.

#### 지원 포맷

| 포맷 | 확장자 | 변환 방식 | 특이사항 |
|------|--------|-----------|----------|
| **Microsoft Word** | .docx | docx-preview 라이브러리 + computed style 인라이닝 | 표, 이미지, 서식 고품질 보존. EMF/WMF 이미지 자동 래스터 변환 |
| **한글** | .hwp | hwp.js 라이브러리 | 한국어 문서 전용 포맷 지원 |
| **새 문서** | - | 빈 HTML 템플릿 생성 | 타임스탬프 기반 고유 ID 부여 |

#### DOCX 변환 상세

DOCX 임포트는 높은 충실도의 HTML 변환을 목표로 합니다.

```
DOCX 파일 업로드
    │
    ▼
docx-preview로 임시 DOM 렌더링
    │
    ▼
모든 요소의 getComputedStyle() 추출
    │
    ▼
인라인 CSS로 변환 (color, fontSize, padding, border 등 전체 스타일 속성)
    │
    ▼
EMF/WMF 이미지 → 래스터 폴백 이미지 추출
    │
    ▼
독립 실행 가능한 HTML 문서 생성
```

### 3.3 AI 문서 수정

자연어 명령을 입력하면 Google Gemini AI가 문서 내용을 자동으로 수정합니다.

#### 기능 상세

| 기능 | 설명 |
|------|------|
| **자연어 수정** | "표를 추가해줘", "3페이지 내용을 요약해줘" 등 자연어 명령으로 문서 수정 |
| **파일 첨부** | Excel, 이미지, PDF, 텍스트 파일을 AI 컨텍스트로 첨부 가능 |
| **글머리 기호 정렬** | 들여쓰기 자동 보정 (•, -, ▪, ○, □ 등) |
| **페이지 나눔 최적화** | 인쇄 시 콘텐츠 분리 방지 CSS 자동 적용 |

#### 첨부 파일 처리

| 파일 유형 | 처리 방식 | AI 전달 형태 |
|-----------|-----------|-------------|
| **Excel (.xlsx/.xls)** | 시트별 CSV 변환 | 텍스트 (프롬프트에 포함) |
| **이미지 (.png/.jpg)** | 1536px 리사이징 + base64 인코딩 | Gemini inlineData (바이너리) |
| **PDF (.pdf)** | base64 인코딩 | Gemini inlineData (바이너리) |
| **텍스트/코드** | 원문 읽기 | 텍스트 (프롬프트에 포함) |

#### 대용량 문서 처리 (핵심 기술)

4MB 이상의 DOCX 임포트 문서도 Gemini API 토큰 한도(1,048,576) 내에서 처리합니다.

```
원본 HTML (4.4MB, ~1.1M 토큰)
    │
    │ ① 인라인 스타일 압축
    │   중복 style="..." → CSS 클래스 (_s0, _s1...) 치환
    │   결과: 74% 크기 감소 (4.4MB → 1.1MB)
    ▼
압축 HTML (1.1MB, ~288K 토큰)
    │
    │ ② base64 이미지 스트립
    │   data:image/... → __B64_IMG_N__ 플레이스홀더 치환
    ▼
경량 HTML
    │
    │ ③ 모드 판별 (임계값: 100KB)
    ├──────────────────┐
    ▼                  ▼
[FULL 모드]        [DIFF 모드]
소형 문서           대형 문서
전체 HTML 반환      변경분만 반환
maxOutput: 32768   maxOutput: 65536
    │                  │
    │                  │ ===DIFF_START===
    │                  │ <<<FIND>>>
    │                  │ (찾을 내용)
    │                  │ <<<REPLACE>>>
    │                  │ (바꿀 내용)
    │                  │ ===DIFF_END===
    │                  │
    ▼                  ▼
    │              Diff 적용 (순차 find/replace)
    │                  │
    ├──────────────────┘
    │
    │ ④ 후처리
    │   CSS 클래스 → 인라인 스타일 복원
    │   __B64_IMG_N__ → base64 이미지 복원
    │   <!--CHART:{...}--> → Chart.js 렌더링
    ▼
최종 수정된 HTML 문서
```

### 3.4 AI 프레젠테이션 생성

HTML 문서를 기반으로 16:9(1280×720px) 프레젠테이션 슬라이드를 자동 생성합니다.

#### 3-모델 파이프라인

3개의 Gemini 모델을 순차적으로 활용하여 고품질 슬라이드를 생성합니다.

```
┌────────────────────────────────────────────────────────────────┐
│                   슬라이드 생성 파이프라인                         │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  [Phase 1] Gemini 2.5 Pro — 콘텐츠 분석                         │
│    ∙ HTML 문서 구조 파악                                        │
│    ∙ 핵심 내용 추출 및 슬라이드 구성 설계                          │
│    ∙ 이미지 생성 프롬프트 추출                                    │
│                                                                │
│  [Phase 2] Gemini 3.1 Flash Image — 이미지 생성                 │
│    ∙ 추출된 프롬프트 기반 이미지 병렬 생성                         │
│    ∙ 배경 이미지: 원본 유지                                      │
│    ∙ 오브젝트 이미지: 배경 자동 제거 (투명화)                      │
│                                                                │
│  [Phase 3] Gemini 2.5 Pro — 슬라이드 HTML 조립                   │
│    ∙ 콘텐츠 + 이미지 조합                                       │
│    ∙ 인라인 CSS 스타일링                                        │
│    ∙ <!--SLIDE_BREAK--> 구분자로 슬라이드 분리                    │
│                                                                │
│  [Phase 4] 뷰포트 자동 보정                                     │
│    ∙ 각 슬라이드 스크린샷 캡처 (1280×720px)                      │
│    ∙ 스크린샷 + HTML을 Gemini Flash에 전송                       │
│    ∙ 넘침(overflow) 감지 시 자동 레이아웃 조정                     │
│    ∙ (폰트 축소, 여백 조정, 콘텐츠 재배치)                        │
│                                                                │
│  [Phase 5] 저장 및 완료                                         │
│    ∙ 이미지 GCS 업로드                                          │
│    ∙ DB 저장 (슬라이드 배열 + 메타데이터)                         │
└────────────────────────────────────────────────────────────────┘
```

#### 생성 중단 복구

생성 중 브라우저 종료 또는 네트워크 오류 시, 페이지 재로드 시점에 미완료 프레젠테이션을
자동 감지하여 중단된 Phase(뷰포트 보정 단계)부터 재개합니다.

### 3.5 슬라이드 편집

생성된 프레젠테이션의 개별 슬라이드 또는 전체 슬라이드를 AI로 수정합니다.

#### 기능 상세

| 기능 | 설명 |
|------|------|
| **개별 수정** | 현재 보고 있는 슬라이드 1장만 자연어 명령으로 수정 |
| **일괄 수정** | 모든 슬라이드에 동일한 명령을 일괄 적용 (예: "전체 배경색을 파란색으로") |
| **비동기 수정** | 수정 중에도 다른 슬라이드 탐색 가능 (Fire-and-Forget 방식) |
| **수정 진행 표시** | 수정 중인 슬라이드에 펄싱 앰버 도트 표시 + 진행 수 배지 |
| **이미지 자동 생성** | 수정 명령에 이미지 관련 내용 포함 시 AI 이미지 자동 생성/삽입 |

#### 슬라이드 네비게이션

| 조작 방식 | 설명 |
|-----------|------|
| **좌/우 화살표 버튼** | 이전/다음 슬라이드 이동 |
| **키보드 방향키** | ArrowLeft / ArrowRight (텍스트 입력 중에는 비활성) |
| **미니 도트 인디케이터** | 클릭하여 특정 슬라이드로 직접 이동. 색상으로 수정 상태 표시 |
| **슬라이드 카운터** | "슬라이드 N / Total" 현재 위치 표시 |

#### 수정 이력 관리

```
[개별 슬라이드 이력]
  슬라이드 1 ── 이력 #1: "제목을 변경해줘"     [되돌리기] [삭제]
              ── 이력 #2: "이미지를 추가해줘"   [되돌리기] [삭제]

[프레젠테이션 스냅샷] (일괄 수정 시 생성)
  스냅샷 #1: "전체 배경을 파란색으로"          [전체 되돌리기]
  스냅샷 #2: "폰트를 나눔고딕으로 통일"        [전체 되돌리기]
```

- **개별 되돌리기**: 특정 슬라이드를 이전 이력 시점으로 복원
- **전체 되돌리기**: 프레젠테이션 전체를 스냅샷 시점으로 복원
- **이력 삭제**: 불필요한 이력 항목 개별 삭제 가능

AI 문서 수정 시 Gemini가 차트 데이터를 출력하면, 클라이언트에서 Chart.js로 렌더링합니다.

#### 지원 차트 유형

| 유형 | 설명 | 용도 예시 |
|------|------|-----------|

| **bar** | 막대 그래프 | 비교 분석, 분포 |
| **pie** | 원형 그래프 | 구성 비율 |
| **doughnut** | 도넛 그래프 | 구성 비율 (변형) |
| **radar** | 레이더 그래프 | 다차원 비교 |

#### 동작 방식

```
Gemini AI 응답에 차트 플레이스홀더 포함
    │
    │  <!--CHART:{"type":"line","title":"매출 추이","labels":["1월","2월",...],
    │   "datasets":[{"label":"매출","data":[100,150,...]}]}-->
    │
    ▼
chartRenderer.js가 플레이스홀더 탐지
    │
    ▼
JSON 파싱 → Chart.js 인스턴스 생성 (오프스크린 캔버스)
    │
    ▼
캔버스 → PNG base64 이미지 변환
    │
    ▼
플레이스홀더를 <img> 태그로 교체 (문서에 이미지 삽입)
```

#### 차트 스타일링

- 10색 팔레트 (blue, red, emerald, amber, violet, pink, cyan, lime, orange, indigo)
- 막대 그래프 배경: 20% 투명도
- 범례: 다중 데이터셋 시 하단 배치
- 축 레이블, 그리드 라인 자동 스타일링

### 3.8 문서 내보내기

편집된 문서를 다양한 포맷으로 내보낼 수 있습니다.

#### 지원 포맷

| 포맷 | 용도 | 처리 방식 |
|------|------|-----------|
| **PNG** | 이미지 공유, 프리뷰 | html2canvas (2배 해상도). 폰트/아이콘 사전 렌더링 |
| **DOCX** | Microsoft Word 편집 | docx 라이브러리로 HTML→Word 변환. 표/이미지/서식 보존 |
| **PDF** | 인쇄, 공식 배포 | jsPDF + html-to-image. 슬라이드별 가로 1페이지. Noto Sans KR 폰트 지원 |
| **HTML** | 웹 게시, 백업 | DOCTYPE + UTF-8 인코딩 완전한 HTML 파일 다운로드 |

#### PNG 내보내기 상세

고품질 PNG 출력을 위해 특수 처리를 수행합니다.

```
① iframe에서 메인 문서로 폰트 복사
   ∙ <link> 스타일시트 복사
   ∙ @font-face 규칙 추출/주입
   ∙ document.fonts.load()로 폰트 프리로드

② Font Awesome 아이콘 사전 렌더링
   ∙ ::before pseudo-element의 content 값 추출
   ∙ 캔버스에 문자 렌더링 (2배 해상도)
   ∙ <img> 태그로 교체 → 캡처 후 원복

③ html2canvas 캡처
   ∙ 2배 스케일, 흰색 배경
   ∙ CORS 허용, tainted canvas 허용

④ IndexedDB 저장 + 자동 다운로드
```

#### PDF 내보내기 상세 (프레젠테이션)

```
① Noto Sans KR Google Font 주입 + 로드 대기
② 슬라이드별 처리:
   ∙ 임시 컨테이너에 슬라이드 HTML 삽입
   ∙ 외부 이미지 URL → base64 data URI 인라이닝 (CORS 처리)
   ∙ toCanvas 캡처 (pixelRatio: 3)
   ∙ jsPDF 페이지 추가 (가로 방향, 1280×720)
   ∙ 슬라이드 간 300ms 렌더링 대기
③ PDF Blob 생성 → 새 탭에서 열기
```

### 3.9 문서 공유

편집된 문서를 공개 링크로 공유할 수 있습니다.

#### 기능 상세

| 기능 | 설명 |
|------|------|
| **공유 링크 생성** | 현재 문서의 HTML 스냅샷을 저장하고 고유 URL 생성 |
| **클립보드 복사** | 생성된 링크 자동 클립보드 복사 (2초간 확인 피드백) |
| **공개 뷰** | 로그인 없이 읽기 전용으로 문서 열람 가능 |
| **에디터에서 열기** | 공유 문서를 편집기에 새 파일로 복사하여 편집 시작 |
| **OG 메타데이터** | 소셜 미디어 공유 시 제목/설명/미리보기 이미지 표시 (SSR) |
| **공유 관리** | 사이드바에서 생성된 공유 링크 목록 확인/삭제 |

#### 공유 흐름

```
[공유 생성]
문서 HTML 스냅샷 ──▶ DB 저장 (shares 테이블) ──▶ 고유 URL 발급 ──▶ 클립보드 복사

[공유 조회]
공유 URL 접속 ──▶ API 서버 ──▶ OG 메타 태그 SSR + iframe 렌더링
                                    │
                                    ▼
                           읽기 전용 HTML 뷰 표시
                                    │
                                    ▼
                        [에디터에서 열기] 버튼 클릭 시
                         → 새 파일 생성 + 편집 모드 전환
```
